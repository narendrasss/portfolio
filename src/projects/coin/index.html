Coin Server

The coin server is a simple API that accepts standard CRUD operations built on the express, mongoose and MongoDB stack. It is tested using Jest and authenticates using JSON web tokens.

Overview

You first have to create an account by registering. The server responds with a token that contains the id of your authenticated user. If you have not authenticated, the server will reject all of your requests.

After registering or logging in, the server allows you to manage three types of objects — transactions, categories, and fixed expenses.

You can query these objects through the api endpoints /tr, /ctg, and /fe respectively. The details of the request/response can be seen in the usage page of the github page.

Features

Other than the standard CRUD operations, the server also supports the following:

  • Protected routes

    All requests to /api/* must have a valid JSON web token attached to its headers. If it doesn't, the server rejects the request with a 401.

  • Query params

    Get requests to /api/tr accepts query parameters such as vendor name, category, maximum amount and before a certain date. Again all details are listed in the usage page on github.
